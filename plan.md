## 1. Project Vision

**VoteWise** is a conversational web assistant that walks first-time and confused voters through the election process — from checking eligibility, registering, knowing deadlines, finding polling locations, to understanding what happens on election day.

### Goals
- ✅ Demonstrate **smart, dynamic assistance** (context-aware responses)
- ✅ Showcase **logical decision-making** (rules engine + LLM hybrid)
- ✅ Make **meaningful use of Google Services** (Cloud Run, Gemini, Cloud Build, Artifact Registry)
- ✅ Be **practical and real-world useful** (real civic value)
- ✅ Stay **clean, maintainable, and accessible**

### Non-Goals
- ❌ Storing personal user data (privacy-first design)
- ❌ Giving political opinions or candidate endorsements
- ❌ Replacing official government election websites

## 3. Solution Overview

### Core User Flows

**Flow 1: Eligibility Check**
```
User: "Can I vote?"
→ Bot asks: age, citizenship, state, current status
→ Rules engine determines eligibility
→ Gemini explains result in plain language
→ Suggests next steps
```

**Flow 2: Registration Guidance**
```
User: "How do I register?"
→ Bot asks state
→ Returns deadlines + methods (online/mail/in-person)
→ Provides official link + step-by-step
```

**Flow 3: Timeline View**
```
User: "What dates should I know?"
→ Returns interactive timeline JSON
→ UI renders milestones (registration close, early voting, election day)
```

**Flow 4: Free-form Q&A**
```
User: "What ID do I need at the polls?"
→ Gemini answers using grounded context from rules engine data
→ Refuses off-topic or partisan questions
```

### Key Differentiator
**Hybrid Rules + LLM architecture** — Election dates and eligibility rules are *deterministic JSON data*, never generated by the LLM. Gemini only handles conversation, summarization, and explanation. This prevents hallucination of critical civic information.

### Data Flow Example: "/chat"

2. FastAPI extracts context, calls Rules Engine for any structured info needed
3. System prompt is built: persona + safety rules + grounded facts from Rules Engine
4. Gemini generates natural reply
5. Response returned to UI; UI renders markdown + suggested follow-up chips

## 9. Antigravity Prompt Playbook

Use these prompts in order inside Antigravity's AI agent. Each builds on the previous.

### Prompt 1 — FastAPI Skeleton
```
Create a FastAPI app in app/main.py with:
- GET /  → serves static/index.html
- POST /chat → accepts {message, context} and returns {reply}
- GET /timeline → returns timeline JSON
- GET /eligibility → query params (age, citizenship, state) returns rules result
- GET /health → returns {"status": "ok"}
Mount /static for static files. Add CORS middleware allowing same-origin.
Load env vars via python-dotenv. Read PORT from env (default 8080).
```

### Prompt 2 — Rules Engine
```
Create app/rules_engine.py with:
- load_data() loads app/data/election_data.json
- check_eligibility(age:int, citizen:bool, state:str) → dict with
  {eligible:bool, reasons:list, next_steps:list}
- get_deadlines(state:str) → dict with registration_deadline,
  early_voting_start, early_voting_end, election_day
- get_state_rules(state:str) → dict with id_required, voting_methods, registration_methods
All functions are pure — no I/O beyond loading the JSON once at import.
Add type hints and docstrings.
```

### Prompt 3 — Election Data
```
Create app/data/election_data.json with sample data for 5 US states
(CA, NY, TX, FL, IL). For each state include:
- registration_deadline (ISO date)
- early_voting_start, early_voting_end, election_day
- id_required (boolean + description)
- voting_methods (array: in-person, mail, early)
- registration_methods (array: online, mail, in-person)
- official_url
Also include a "general_faqs" array with 10 common election questions and answers.
Mark all data clearly as illustrative/demo with a "_disclaimer" field.
```

### Prompt 4 — Gemini Client
```
Create app/gemini_client.py:
- Reads GEMINI_API_KEY from env
- Uses model "gemini-2.5-flash"
- Function generate_reply(user_message:str, grounded_context:dict) -> str
- System prompt MUST:
  1. Identify as VoteWise, an election process assistant
  2. Refuse partisan questions, candidate endorsements, voter persuasion
  3. Refuse off-topic questions politely
  4. Always cite that data shown is illustrative; refer users to official sources
  5. Use the grounded_context dict as the source of truth for any dates/rules
- Wrap API calls in try/except; return user-friendly error message on failure
- Add request timeout of 15 seconds
```

### Prompt 5 — Prompts Module
```
Create app/prompts.py containing:
- SYSTEM_PROMPT: the full system prompt string for VoteWise
- build_chat_prompt(user_message, context_dict) -> str
- REFUSAL_TEMPLATES: dict of canned refusals for partisan/off-topic queries
Keep prompts concise and well-commented.
```

### Prompt 6 — Pydantic Models
```
Create app/models.py with Pydantic v2 models:
- ChatRequest: message:str (1-1000 chars), context:Optional[dict]
- ChatResponse: reply:str, suggested_followups:list[str]
- EligibilityRequest: age:int (0-150), citizen:bool, state:str (2-letter)
- EligibilityResponse: eligible:bool, reasons:list[str], next_steps:list[str]
Add validators for state (must be 2 uppercase letters).
```

### Prompt 7 — Frontend Chat UI
```
Create static/index.html, static/style.css, static/app.js:
```

## 10. Deployment Plan

### Deploy Command (single command, source-based)
```bash
gcloud run deploy votewise \
  --source . \
  --region us-central1 \
  --allow-unauthenticated \
  --set-env-vars GEMINI_API_KEY=YOUR_KEY_HERE \
  --min-instances 0 \
  --max-instances 2 \
  --memory 512Mi \
  --cpu 1 \
  --port 8080 \
  --timeout 60s
```

### What Happens
1. Cloud Build packages your source
2. Buildpacks (or your Dockerfile) builds the image
3. Image pushed to Artifact Registry
4. Cloud Run deploys it
5. You get a public HTTPS URL

### Post-Deploy Verification
```bash
# Smoke test
curl https://YOUR-URL/health

# Test chat
curl -X POST https://YOUR-URL/chat \
  -H "Content-Type: application/json" \
  -d '{"message":"Am I eligible to vote in California at age 19?"}'
```

### Rollback Plan
```bash
gcloud run revisions list --service votewise --region us-central1
gcloud run services update-traffic votewise \
  --to-revisions REVISION_NAME=100 --region us-central1
```
